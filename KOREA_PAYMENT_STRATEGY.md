# MNI ACV - 대한민국 결제 및 회원가입 시스템 전략

## 1. 결제 시스템 분석

### 1.1 Stripe vs 국내 결제 서비스 비교

| 항목 | Stripe | 토스페이먼츠 | 아임포트 | 나이스페이 |
|------|--------|-----------|---------|----------|
| **한국 사업자 지원** | ❌ 제한적 | ✅ 완벽 지원 | ✅ 완벽 지원 | ✅ 완벽 지원 |
| **개인사업자 등록** | ❌ 불가 | ✅ 가능 | ✅ 가능 | ✅ 가능 |
| **법인 등록** | ❌ 어려움 | ✅ 가능 | ✅ 가능 | ✅ 가능 |
| **국내 카드 결제** | ⚠️ 제한적 | ✅ 완벽 | ✅ 완벽 | ✅ 완벽 |
| **계좌이체** | ❌ 없음 | ✅ 지원 | ✅ 지원 | ✅ 지원 |
| **휴대폰 결제** | ❌ 없음 | ✅ 지원 | ✅ 지원 | ✅ 지원 |
| **API 문서** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| **개발자 경험** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| **수수료** | 2.9% + $0.30 | 2.5~3.5% | 2.5~3.5% | 2.5~3.5% |
| **정산 주기** | 2일 | 다음날 | 다음날 | 다음날 |

### 1.2 추천 방안: **토스페이먼츠 (Toss Payments)**

**선택 이유:**
- **개인사업자/법인 모두 완벽 지원**: 사업자등록증만 있으면 가입 가능
- **국내 결제 수단 완벽 지원**: 신용카드, 계좌이체, 휴대폰 결제, 간편결제(카카오페이, 네이버페이 등)
- **빠른 정산**: 다음날 정산으로 현금흐름 관리 용이
- **개발자 친화적**: Node.js SDK 지원, 명확한 문서
- **낮은 수수료**: 2.5~3.5% (Stripe보다 저렴)
- **한국 스타트업 생태계**: 한국 기업이므로 한국식 세금 계산, 정산 시스템 최적화

## 2. 회원가입 및 인증 시스템

### 2.1 현재 상태
- Manus OAuth 통합 준비 완료
- 기본 회원 테이블 구조 설계됨

### 2.2 추천 방안: **Manus OAuth + 국내 인증 추가**

**이유:**
- Manus OAuth는 기본 인증 제공하지만, 한국 사용자 경험 최적화 필요
- 국내 사용자들은 소셜 로그인 선호 (카카오, 네이버, 구글)
- 멤버십/구독 관리 시 본인인증 필요 (PASS, 카카오 본인인증 등)

**구현 방안:**
```
1단계: Manus OAuth (기본 인증)
2단계: 카카오/네이버 소셜 로그인 추가
3단계: 선택적 본인인증 (휴대폰 인증, PASS)
```

## 3. 멤버십 시스템 설계

### 3.1 멤버십 구조 제안

| 등급 | 월 가격 | 혜택 |
|------|--------|------|
| **Free** | 무료 | 갤러리 열람, 뉴스레터 구독 |
| **Member** | ₩4,900 | Drop 사전 공개, 10% 할인, 배송료 무료 |
| **VIP** | ₩9,900 | 모든 Member 혜택 + 한정판 상품 조기 구매, 20% 할인 |

### 3.2 구현 기술
- **구독 관리**: 토스페이먼츠 정기결제 (Billing API)
- **데이터베이스**: 멤버십 테이블, 구독 이력 테이블
- **자동화**: Webhook으로 결제 성공/실패 처리

## 4. 세금 및 법무 고려사항

### 4.1 개인사업자 기준
- **부가가치세**: 매출 8,000만원 이상 시 필수 등록
- **소득세**: 순이익에 대해 연 1회 신고 (5월)
- **정산 세금**: 토스페이먼츠에서 자동 계산 (3.3% 세금 선입금)

### 4.2 법인 기준
- **부가가치세**: 매출 8,000만원 이상 필수 등록
- **법인세**: 순이익에 대해 분기별 신고
- **원천징수**: 해당 없음 (B2C 거래)

### 4.3 필수 준비물
- 사업자등록증 (개인/법인)
- 통장 (정산용)
- 사업자 이메일
- 휴대폰 (본인인증용)

## 5. 구현 로드맵

### Phase 1: 토스페이먼츠 기본 결제 (1-2주)
```
- 토스페이먼츠 가입 및 API 키 발급
- 결제 페이지 UI 구현
- 주문 데이터베이스 테이블 설계
- 결제 완료/실패 처리 로직
```

### Phase 2: 회원가입 및 인증 (1-2주)
```
- 카카오/네이버 소셜 로그인 추가
- 회원 프로필 페이지
- 주문 이력 조회 기능
```

### Phase 3: 멤버십 시스템 (2-3주)
```
- 멤버십 등급 관리
- 정기결제 (구독) 기능
- 멤버십 혜택 적용 로직
- 자동 갱신/취소 처리
```

### Phase 4: 관리자 대시보드 (2-3주)
```
- 주문 관리 (조회, 배송 상태 변경)
- 매출 통계 및 분석
- 회원 관리
- 멤버십 통계
```

## 6. 유지보수 및 운영 가이드

### 6.1 일일 운영
- **아침**: 전날 주문 확인 및 배송 준비
- **오후**: 결제 실패 건 모니터링 및 고객 연락

### 6.2 주간 운영
- 매출 리포트 검토
- 고객 문의 응답
- 인벤토리 관리

### 6.3 월간 운영
- 토스페이먼츠 정산 확인
- 세금 계산 및 기록
- 멤버십 갱신 현황 분석

### 6.4 분기별 운영
- 세금 신고 준비 (개인사업자: 5월, 법인: 분기별)
- 사업 성과 분석
- 마케팅 전략 수립

## 7. 예상 비용 및 수익성

### 7.1 초기 비용
- 도메인: ₩10,000/년
- 호스팅 (Manus): 무료~₩100,000/월
- 토스페이먼츠 가입: 무료

### 7.2 거래당 비용
- 토스페이먼츠 수수료: 2.5~3.5%
- 배송료: ₩3,000~5,000 (사업자 부담)

### 7.3 손익분기점 예시
```
제품 가격: ₩80,000 (TOILET PAPER TEE)
수수료: ₩2,400 (3%)
배송료: ₩4,000
순이익: ₩73,600

월 10개 판매 시: ₩736,000
월 50개 판매 시: ₩3,680,000
```

## 8. 최종 추천사항

### ✅ 즉시 시작 가능
1. **토스페이먼츠 가입**: 사업자등록증 준비 후 5분 내 가입 가능
2. **기본 결제 기능**: 2주 내 구현 가능 (SDK 제공)
3. **주문 관리 시스템**: 간단한 대시보드로 시작

### ⚠️ 주의사항
- **세금 신고**: 반드시 세무사와 상담 (초기 ₩50,000~100,000)
- **약관 및 개인정보보호**: 법무사 검토 권장 (초기 ₩100,000~200,000)
- **배송 파트너**: 한국 택배사와 계약 필수 (CJ, 로젠, 우체국 등)

### 🎯 성공 전략
1. **MVP 출시**: 기본 결제 + 회원가입만으로 시작
2. **고객 피드백**: 초기 고객 50명 기준으로 개선
3. **점진적 확장**: 멤버십 → 구독 → 마케팅 자동화 순서로 확대
